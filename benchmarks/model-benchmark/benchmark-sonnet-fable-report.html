Observations by Test

1. Long-Context Memory (20%)

Both models correctly answered all six questions. Fable 5 was marginally faster (11.47s vs 12.38s) and slightly more thorough, adding a note about the hard-coded cache TTL. Sonnet 5's answer was more concise. Cost gap is the story: Sonnet 5 cost $0.0282 vs Fable 5's $0.1142 (4×).

2. Multi-Step Reasoning (20%)

Both produced solid cascading-failure analyses. Sonnet 5 finished in 45.28s, Fable 5 in 50.06s. Fable 5's output included a useful ASCII flow diagram; Sonnet 5's was more compact. Again, Sonnet 5 was ~3.7× cheaper for comparable quality.

3. Code Generation (20%)

Both generated complete, production-style Redis + Lua sliding-window rate limiters. Sonnet 5 was faster (42.97s vs 48.13s) and cheaper. Fable 5's module opened with strong design notes. Neither contained placeholders.

4. Instruction Following (15%)

Sonnet 5 was meaningfully faster here: 33.30s vs 45.68s, and still 4× cheaper. This task rewards literal constraint adherence; both models attempted the format, but Sonnet 5 delivered more quickly.

5. Sustained Attention (10%)

The prompt referenced files that were not actually included in context. Sonnet 5 correctly flagged the missing content and asked for the files — a desirable "I don't know" response. Fable 5 returned a near-empty 4-token response in only 4.45s, which is unhelpful. Sonnet 5 wins on usefulness despite being slower on the clock.

Note: The "Sustained Attention" prompt was intentionally underspecified (file paths without contents). Sonnet 5 handled the ambiguity honestly; Fable 5 did not engage.

Recommendations for OpenClaw

  • Primary model for code & agent tasks: Claude Sonnet 5 — faster on 3 of 5 tests, 4× cheaper than Fable 5, and fully capable.
  • Fallback for very long outputs: Claude Fable 5 — only when the 128K output ceiling or a reasoning-heavy second opinion justifies the 4× premium.
  • Cost-sensitive work: Sonnet 5 wins across the board; even vs. other Venice models, it is competitive on speed.
  • When to avoid Fable 5: Standard code generation, instruction-following, and routine long-context retrieval are not worth the extra cost.

Methodology

Tests were executed through the Venice AI chat completions API with identical prompts and max_tokens=4096. Each test was run once per model (no retries). Costs were computed from actual prompt and completion token counts using the per-model pricing published on Venice AI. Timing is end-to-end latency from request to first complete response.
