How we score

Every benchmark is run with fixed prompts, default sampling, and transparent heuristics. Scores are reproducible and auditable.

1 · Intent Understanding

Models receive ambiguous, underspecified prompts. Points for clarifying questions, structured proposals, and explicit uncertainty before assuming details.

2 · One-Shot UI Generation

Models must emit a single self-contained HTML file. Scored on visual polish, semantic structure, accessibility, responsiveness, and whether it runs without edits.

3 · Brick Breaker Realism

Models generate a playable Canvas game in one shot. Scored on physics plausibility, control responsiveness, collision detection, game states, particle/sound effects, and responsive design.

4 · Startup in a Weekend

Models receive a tight MVP brief for an AI contract-review SaaS. Points for system architecture, concrete database schema, API endpoints, auth strategy, AI pipeline, deployment stack, day-by-day schedule, cost estimate, risk/mitigations, and explicit out-of-scope boundaries.

5 · VivIndex Composite

Weighted average of four tracks. Startup in a Weekend 35%, Intent Understanding 25%, One-Shot UI 20%, Brick Breaker 20%. Missing benchmarks are labelled and weights are renormalized rather than scored as zero.

6 · Full protocol

Exact prompts, scoring math, Arcade design, and non-claims are published on the Experimental Design page — the lab notebook for this suite.
